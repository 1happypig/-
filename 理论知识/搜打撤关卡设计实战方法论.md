# 搜打撤关卡设计实战方法论

> 本文以 **搜打撤品类关卡策划** 的视角出发，聚焦于地图设计的具体工作流程、设计决策和实战方法，而非品类宏观分析。
>
> 适合场景：做地图拆解时对照参考、面试关卡策划岗位时展示专业度、日常设计工作中查阅。
>
> 综合整理自 GameRes、机核 GCORES、The Level Design Book、SMU Guildhall、CGMA、行业岗位要求等来源。

---

## 一、搜打撤关卡策划是做什么的

### 1. 岗位画像

搜打撤关卡策划与一般关卡策划的核心区别在于：**你的设计需要同时服务于 [[专业词汇注解/PvP|PvP]] 对抗、[[专业词汇注解/PvE|PvE]] 挑战和局内外经济循环**。一个典型的搜打撤关卡策划日常工作包括：

| 工作内容 | 占比 | 说明 |
|---------|------|------|
| 地图 Layout 设计 | 25% | 2D 布局图，标记 [[专业词汇注解/POI|POI]]/动线/出生点/撤离点 |
| 白盒搭建与迭代 | 30% | UE5 灰盒搭建 → 内部跑测 → 修改 → 再跑测 |
| 战斗空间调优 | 20% | 掩体位置、枪线长度、垂直位、刷怪点 |
| 数据与反馈驱动 | 15% | 热力图、撤离率、死亡率、玩家反馈 |
| 美术需求对接 | 10% | 将白盒需求交接给地编/场景美术 |

### 2. 搜打撤关卡策划的核心能力

根据行业岗位要求（腾讯、网易等搜打撤项目），核心能力排序：

1. **空间设计能力**：能从 2D Layout 推导出 3D 空间体验，理解尺度、比例、动线
2. **搜打撤品类理解**：深刻理解搜→打→撤三阶段的体验差异，能针对性地进行空间设计
3. **白盒快速迭代**：熟练使用 [[专业词汇注解/UE5|UE5]] 等引擎搭建可玩原型，快速验证设计假设
4. **竞品分析能力**：能拆解《三角洲》《暗区突围》《塔科夫》的地图设计并提炼方法论
5. **数据驱动意识**：理解热力图、击杀位置、撤离率等数据如何指导设计迭代

加分项：
- 搜打撤游戏深度体验
- 有 UE5 白盒作品
- 会写简单脚本/[[专业词汇注解/蓝图|蓝图]]

## 二、搜打撤地图设计完整流程

这是从零开始设计一张搜打撤地图的标准化流程，也是面试中展示专业度的最佳框架。

### 阶段 1：前期规划（[[专业词汇注解/原型|Pre-production]]）

**目标**：确定地图的体验定位和核心参数

#### 1.1 定义地图杠杆度

每张搜打撤地图都有一个"[[专业词汇注解/杠杆度|杠杆度]]"——它服务于高风险高回报还是低风险低回报？

| 杠杆度 | 定位 | 设计表现 | 案例 |
|-------|------|---------|------|
| 低杠杆（新手图） | 积累经济，建立信心 | 出生点安全、撤离点多、高级物资少 | 三角洲·零号大坝 |
| 中杠杆（标准图） | 常规体验，平衡风险收益 | 适中 POI 密度、多条可选路线 | 暗区·山谷 |
| 高杠杆（高难图） | 消耗经济，高手对决 | 出生点密集、高价值 POI 集中、撤离点苛刻 | 三角洲·航天基地 |

#### 1.2 确定地图参数

| 参数 | 决策依据 | 影响 |
| ---- | -------- | ---- |
| 地图尺寸 | 预期对局时长 × 玩家机动速度 | 太小则遭遇频繁，太大则跑图时间长 |
| 玩家人数 | 单人/双人/三人队 | 决定 POI 规模和建筑尺度 |
| 对局时长 | 15min / 25min / 40min | 决定地图面积和资源总量 |
| 主题/风格 | 室内/室外/混合 | 决定垂直维度、枪线长度分布 |

### 阶段 2：Layout 设计（2D 布局）

**目标**：绘制俯视布局图，标注所有关键区域和[[专业词汇注解/动线|动线]]

#### 2.1 布局设计的五个要素

```
┌──────────────────────────────────────────────┐
│  [第一步]                                     │
│  放出生点 → 放撤离点 → 放高价值POI → 连接动线   │
│                                              │
│  复活点A ──╲                                  │
│              ╲──── 高价值POI ──── 复活点C       │
│  复活点B ──╱         │                        │
│                      │                        │
│                   撤退点1                     │
│                   撤退点2                     │
└──────────────────────────────────────────────┘
```

**步骤详解**：

**(1) 布置[[专业词汇注解/出生点|出生点]]（Spawn Points）**
- 数量：一般为玩家人数 × 1.5 ~ 2 倍（如 8 队地图配 12-16 个出生点）
- 分布原则：**固定位置，随机刷新**——玩家在进局前不知道自己的出生点
- 间隔：相邻出生点之间应有足够距离，避免"出生遭遇"
- 补救机制：对于位置较差的出生点，提供替代路线或早动权补偿（如航天基地的"刷卡买路"机制）

**(2) 布置[[专业词汇注解/撤离点|撤离点]]（Extract Points）**
- 数量：随着难度递增而减少（低难度 4-6 个 → 高难度 2-3 个）
- 类型组合：常规边缘撤离点 + 条件撤离点（[[专业词汇注解/拉闸撤离|拉闸]]/任务/付费）
- 位置原则：
  - 常规撤离点位于地图边缘，迫使玩家横穿地图产生遭遇
  - 条件撤离点位于核心区附近，信息风险高（拉闸会通报全图）
- 时间窗口：等待撤离的时间就是战斗窗口（越长风险越大）

**(3) 布置高价值 POI**
- 数量：全图 3-5 个核心 POI，分布在多条动线的交汇处
- 层级结构：1-2 个顶级 POI（如航天基地核心区）+ 若干次级 POI
- 引力平衡：避免单一 POI 吸引力过强导致玩法固化（如暗区[[专业词汇注解/POI引力陷阱|汽车旅馆综合征]]）

**(4) 连接动线**
- 主干道：连接出生点 → [[专业词汇注解/POI|POI]] → 撤离点，形成主要博弈走廊
- [[专业词汇注解/侧翼路|侧翼路]]：提供绕行选项，避免主干道被狙击完全封锁
- 死区/过渡带：低物资、低掩体区域，迫使玩家小心通过或绕行

**(5) 标注关键节点**
- [[专业词汇注解/Choke点|Choke]] 点（动线收束处，战斗高发区）
- 视野控制位（长枪优势位/[[专业词汇注解/枪线|信息位]]）
- [[专业词汇注解/垂直转换点|垂直转换点]]（室内←→屋顶/地下室转换）

#### 2.2 规划动线的三种模式

搜打撤地图的动线在不同阶段有不同的核心关系：

| 阶段 | 动线关系 | 设计目标 |
|------|---------|---------|
| 出生点→搜索点 | [[专业词汇注解/先打后搜|先打后搜]] / [[专业词汇注解/先搜后打|先搜后打]] | 决定早期节奏，控制首次接敌时间 |
| 搜索点→搜索点 | 集中连接 / 分散间隔 | 控制战斗频率，决定中期遭遇概率 |
| 搜索点→撤离点 | 边缘撤离 / 条件撤离 | 决定后期收束方式和最终博弈强度 |

**先打后搜 vs 先搜后打**：
- **先打后搜**（如地铁逃生）：出生点到核心区之间通过 [[专业词汇注解/Choke点|Choke]] 连接，让喜欢打架的玩家早期接敌，同时给避战玩家绕行的空间
- **先搜后打**（如三角洲模式）：出生点决定谁能优先接触资源，[[专业词汇注解/安全期|安全期]]较长，战斗更多发生在资源区内

### 阶段 3：白盒搭建（[[专业词汇注解/白盒|Blockout]]）

**目标**：将 2D Layout 转化为 3D 可游玩原型

#### 3.1 白盒搭建的标准步骤

```
手绘草图 → 地平面+比例物 → 墙体搭建 → 开门窗 → 放置掩体 → 内部跑测 → 修改 → 再跑测
```

**关键要点**：

1. **比例感**：先放一个人形[[专业词汇注解/原型|比例模型]]（1.7-1.8m），所有尺寸以此为标准
2. **墙体高度**：约为比例物高度的 1.5-2 倍
3. **通道宽度**：入口和走廊至少为比例物宽度的两倍
4. **色彩区分**：地面用灰色，墙面用浅色，帮助视觉区分
5. **掩体用色**：[[专业词汇注解/全掩体|全掩体]]用深色，[[专业词汇注解/半掩体|半掩体]]用中间色，区分明显

#### 3.2 [[专业词汇注解/白盒|白盒]]阶段的引导设计（顽皮狗方法）

搜打撤地图的引导需要比传统 [[专业词汇注解/FPS|FPS]] 更含蓄——你不想让地图"太好认"，但也不能让玩家迷路到失去耐心。

| 引导手段 | 搜打撤中的应用 |
| -------- | -------------- |
| **可供性（Affordance）** | 物体的形状暗示用途。梯子→可攀爬，门把手→可开 |
| **[[专业词汇注解/地标|地标]]（Landmark）** | 独特的天际线轮廓，帮助定位。如塔科夫的厂房红顶 |
| **[[专业词汇注解/否认可供性|否认可供性]]** | 明确的障碍物告诉玩家"此路不通"，避免无用探索 |
| **[[专业词汇注解/引导线|引导线]]** | 道路、管道、电缆延伸引导视线至目标 |
| **[[专业词汇注解/空间压缩|空间压缩]]** | 窄道/障碍物引导玩家走向指定路线 |
| **开口与缺口** | 门、拱门吸引进入，制造好奇 |
| **[[专业词汇注解/面包屑引导|面包屑]]** | 物资、尸体、血迹逐步引导玩家深入 |

#### 3.3 白盒常见问题

| 问题 | 表现 | 解决方法 |
| ---- | ---- | -------- |
| 空间过大 | 跑图时间长，遭遇稀少 | 用比例物感知尺度，缩小无效空间 |
| 空间过小 | 巷战频繁，被堵在一个区域 | 扩大关键走廊，增加绕行路线 |
| 迷路/动线不清 | 玩家停在路口犹豫 | 增加地标或空间压缩引导 |
| Choke 过于狭窄 | 变成"谁先露头谁死" | 增加迂回路线或投掷物空间 |
| 掩体分布不均 | 一方优势过大 | 跑测后根据死亡位置调整 |

### 阶段 4：调优与迭代

**目标**：通过测试和数据反复打磨地图体验

#### 4.1 迭代关注的核心指标

| 指标 | 健康范围 | 如果异常 |
|------|---------|---------|
| [[专业词汇注解/撤离率|撤离率]] | 30%-50% | 太高→压力不够；太低→挫败过强 |
| 平均存活时间 | 地图预期时长的 60-70% | 太短→出生杀；太长→避战过度 |
| 各出生点撤离率 | 标准差 < 10% | 某个出生点胜率过低→需要补强 |
| 核心 POI 访问率 | 60-80% 玩家会经过 | 太低→POI 位置不佳；太高→其他区域浪费 |
| 平均击杀数/局 | 1.5-3 | 太少→战斗不足；太多→过于拥挤 |

#### 4.2 数据驱动的调优方法

- **[[专业词汇注解/热力图|热力图]]**：查看死亡/击杀位置，发现异常热点或死区
- **动线追踪**：分析玩家实际路线 vs 设计预期路线
- **[[专业词汇注解/分区留存率|分区留存率]]**：哪些区域玩家停留时长异常（过高 → 太安全，过低 → 直接绕开）
- **[[专业词汇注解/全队覆灭率|全队覆灭率]]**：在某个区域全队阵亡的比例（过高 → 设计问题）

## 三、搜打撤地图的节点设计手册

这一节是 **关卡策划最实用的工具集**——每个[[专业词汇注解/节点|节点]]类型的设计要点都在这里。

### 1. 出生点设计规范

| 维度 | 要求 |
|------|------|
| 数量 | 玩家人数 × 1.5 ~ 2 倍 |
| 间隔 | 与最近的其他出生点 ≥ 20 秒路程 |
| 朝向 | 面对出口/主路线方向，避免面壁 |
| 掩体 | 有基本掩体覆盖，避免被远处狙击 |
| 出口 | 至少 2 条不同方向的出口路线 |
| 补偿 | 劣势出生点需要补偿机制（如更近的物资点） |

### 2. POI 设计规范

每个 POI 需要明确三个属性：

```
POI = 潜在收益 × 风险系数 × 交通成本
```

| POI 类型 | 收益 | 风险 | 交通成本 | 密度 |
|---------|------|------|---------|------|
| 顶级 POI | 极高（钥匙房/保险箱） | 极高（位于动线交汇处） | 高 | 全图 1-2 个 |
| 次级 POI | 中高（高级容器） | 中高（需侧翼警戒） | 中 | 全图 3-5 个 |
| 分散点 | 低-中（普通容器） | 低（偏离主动线） | 低-中 | 全图 8-15 个 |

**[[专业词汇注解/POI引力陷阱|POI 引力陷阱]]**：如果一个 POI 的引力过强（收益远超风险+交通成本），所有玩家都会涌向它，导致其他区域被浪费。解决方案：
- 调整该 POI 的风险（增加动线交汇、增加防守位）
- 上调其他 POI 的收益（保持全图引力分散）
- 使用屏障机制（钥匙/任务）控制进入节奏

### 3. Choke 点设计规范

Choke 点是动线收束处，也是最易爆发战斗的区域。设计原则：

| 原则 | 说明 |
|------|------|
| 双方距离对等 | 从不同方向到达 Choke 的时间应尽量相等 |
| 侧翼支持 | Choke 应有至少一条侧翼路线，避免被单点压制 |
| 投掷物空间 | 足够让玩家使用手雷/闪光弹开路 |
| 掩体平衡 | 到达方的掩体 + 防守方的掩体应相对平衡 |
| 功能明确 | 玩家清楚知道"这是个危险区域，要小心" |

### 4. [[专业词汇注解/撤离点|撤离点]]设计规范

| 类型 | 数量 | 位置 | 条件 | 设计要点 |
|------|------|------|------|---------|
| 常规撤离 | 2-4 | 地图边缘 | 无，直接等待撤离 | 需要等待 5-10 秒，暴露位置 |
| 随机撤离 | 1-2 | 每局随机刷新 | 随机位置触发 | 打破固定路径依赖 |
| 拉闸撤离 | 1-2 | 核心区附近 | 激活拉闸按钮 | 全图广播"某处正在撤离"，信息风险极高 |
| 条件撤离 | 1-2 | 指定位置 | 丢弃背包/付费/合作 | 提供不同策略选择 |

**撤离点设计核心**：撤离点不是为了让你轻松走掉，而是**制造最后的博弈空间**。一个好的撤离点让玩家面临选择：
- 直接冲过去撤离（高风险，可能被堵）
- 先清场再撤离（安全，但可能错过撤离窗口）
- 换另一个更安全的撤离点（绕路，路上可能遭遇）

### 5. 掩体设计规范

搜打撤的[[专业词汇注解/掩体|掩体]]设计与传统竞技场 FPS 的区别在于：**掩体不是为了创造公平对枪，而是为了创造战术选择**。

| 掩体类型 | 防护 | 搜打撤中的特殊用法 |
|---------|------|-------------------|
| [[专业词汇注解/全掩体|全掩体]]（Full Cover） | 安全躲避 | 打包/换弹/躲侦查 |
| [[专业词汇注解/半掩体|半掩体]]（Half Cover） | 部分遮挡 | 架枪/观察/对枪 |
| [[专业词汇注解/可穿透掩体|可穿透掩体]] | 低 | 信息博弈（打穿判断有人/没人） |
| 高位掩体 | 视野优势 | 控制关键走廊 |

**掩体布置检查清单**：
- [ ] 每个主要交战区都有至少 2 个掩体群
- [ ] 掩体之间间距合理，不会让玩家跑"空旷区"
- [ ] 存在可穿透掩体，供信息博弈
- [ ] 高位和低位之间有视野关系
- [ ] 掩体不会完全阻挡一条路径（至少留侧身空间）

## 四、经典[[专业词汇注解/关卡策划|设计理论]]工具集

> 这一节汇集了关卡设计领域最经典的**前人方法论**，每个理论都附上"在搜打撤设计中的落地用法"。

### 1. 凯文·林奇《城市意象》五要素

**来源**：Kevin Lynch, *The Image of the City* (1960)，城市设计经典。

**核心内容**：人在头脑中构建"心智地图"来认知空间，依赖五个要素：

```
地标（Landmarks）—— 独特的参照物，从远处即可识别
路径（Paths）—— 移动的通道，有方向性
边界（Edges）—— 线性分隔，划分不同区域
区域（Districts）—— 有统一特征的面状空间
节点（Nodes）—— 路径交汇处，决策/汇聚点
```

**在搜打撤中的落地**：

| 要素 | 搜打撤中的应用 | 具体案例 |
| ---- | -------------- | -------- |
| 地标 | 帮助玩家定位的独特建筑/地形 | 塔科夫海关的厂房红顶，三角洲航天基地的发射塔 |
| 路径 | 玩家选择的行进路线，区分主/侧/战术路 | 出生点到 POI 再到撤离点的动线网络 |
| 边界 | 地图的外边界 + 内部危险区的分隔 | 地图边界的撤离点方向，切尔诺贝利核电站外围辐射区 |
| 区域 | 每个 POI 应有独特的视觉/功能身份 | 暗区山谷的别墅区 vs 海滨别墅 vs 港口 |
| 节点 | 出生点/Choke/撤离点 | 玩家做决策的位置，如十字路口、拉闸点 |

**设计应用**："在做 Layout 时，我会参考凯文·林奇的五要素来确保地图的可读性——每个 POI 都是一个有独特身份的区域，通过地标帮助玩家定位，通过路径组织动线。"

### 2. 克里斯托弗·亚历山大的《模式语言》

**来源**：Christopher Alexander, *A Pattern Language* (1977)，253 个建筑模式构成的设计语言。

**核心内容**：每个模式描述一个"在某种上下文中反复出现的问题及其解决方案"。适用于关卡设计的经典模式包括：

| 模式 | 原始定义 | 关卡设计中的映射 |
| ---- | -------- | ---------------- |
| **路径与目标（Paths and Goals）** | 地标之间应保持互见距离 | 玩家始终能看到下一个目标点，但需要走过去才能到达 |
| **高地（High Places）** | 高处可俯瞰环境 | 狙击位、观察位、信息位 |
| **私密梯度（Intimacy Gradient）** | 从公共到私密的过渡层次 | 从开阔地带到室内再到密室的空间递进 |
| **活动节点（Activity Nodes）** | 节点之间应有步行可达的距离 | POI 之间的间距应与对局时长匹配 |
| **可到达的绿色（Accessible Green）** | 每片绿地都应可达 | 避免"看得见去不了"的无效空间 |

**在搜打撤中的落地**：

搜打撤地图中的"私密梯度"极为重要——从出生点（完全安全）到主干道（可能遭遇）到高价值 POI 内部（高度危险）到撤离点（最后博弈），每个空间层次的安全感是递进的，玩家的警觉度也应随之调整。

如果安全梯度跳跃太大（比如出生点直接暴露在狙击线下），玩家会感到不公平。

### 3. [[专业词汇注解/心流理论|心流理论]]（Flow Theory）

**来源**：Mihaly Csikszentmihalyi (1990)。玩家完全沉浸于活动中的心理状态。

**核心内容**：心流发生的条件是**挑战与技能动态平衡**：

```
        高
         │  焦虑 ←──────── 心流通道 ────────→ 无聊
玩家挑战 │     ↘        ↙        ↙
         │       ↘    ↙      ↙
         │         ↘  ↙    ↙
         │           ↘  ↙
         │             ←── 技能增长方向
         └──────────────────────────────→
                    玩家技能        高
```

**在搜打撤中的落地**：

搜打撤的独特之处在于：**玩家可以自己调节挑战水平**。新手可以选择"[[专业词汇注解/老鼠|老鼠]]路线"避战搜刮（降低挑战），高手可以选择直冲核心 POI（提高挑战）。关卡设计需要为不同技能水平的玩家**同时提供选择**：

| 玩家类型 | 选择 | 关卡设计对应 |
|---------|------|------------|
| 新手 | 避战，边缘搜刮 | 侧翼路线+低级容器+简单撤离 |
| 中级 | 适度战斗，选择性搜刮 | 标准路线+中等 POI+常规撤离 |
| 高手 | 主动战斗，抢夺核心资源 | 核心 POI+高密度 Choke+条件撤离 |

**设计应用**："搜打撤的一个设计优势是玩家能自调节心流——关卡设计的关键是提供不同杠杆度的选项，让不同水平的玩家都能找到适合自己的心流通道。"

### 4. [[专业词汇注解/MDA框架|MDA 框架]]（机制-动态-美学）

**来源**：Robin Hunicke, Marc LeBlanc, Robert Zubek, GDC 2004 发表。

**核心内容**：游戏设计拆解为三个层次，设计师与玩家的视角相反：

```
设计师视角： 机制（Mechanics）→ 动态（Dynamics）→ 美学（Aesthetics）
玩家视角：   美学（Aesthetics）→ 动态（Dynamics）→ 机制（Mechanics）
```

| 层次 | 定义 | 搜打撤示例 |
|------|------|-----------|
| 机制 | 规则、数据、算法 | 死亡掉落所有非[[专业词汇注解/安全箱|安全箱]]物品、撤离需要倒计时 10 秒 |
| 动态 | 玩家与机制的互动行为 | 玩家因为怕损失而选择避战搜刮，形成"[[专业词汇注解/老鼠|老鼠]]"流派 |
| 美学 | 玩家的情感体验 | 成功撤离时的如释重负→狂喜，死亡时的挫败→想再来一局 |

**在搜打撤中的落地**：

MDA 框架帮助关卡设计师理解：**你设计的是机制（规则/空间），但玩家体验的是美学（情感）**。当你调整一个出生点的位置时：
- **机制层面**：你移动了空间中的一个点
- **动态层面**：玩家到达核心 POI 的时间提前了 10 秒，某些路线变得更有价值
- **美学层面**：某些玩家觉得公平性提高了，某些玩家觉得"怎么总是我刷在烂位置"

### 5. 克里斯托弗·托顿的[[专业词汇注解/图底关系|建筑学关卡设计方法]]

**来源**：Christopher Totten, *An Architectural Approach to Level Design*。

**核心内容**：系统建立建筑学与关卡设计的跨学科理论体系。核心概念包括：

| 概念 | 定义 | 关卡设计中的用法 |
|------|------|----------------|
| **[[专业词汇注解/图底关系|图底关系]]（Figure-Ground）** | 建筑实体与开放空间的关系 | 关卡中"可站立的平台"与"空隙/深渊"的关系 |
| **形式与虚空（Form-Void）** | 3D 空间的体块切割 | 室内空间是"挖空"的建筑体块 |
| **到达感（Arrival）** | 通过空间对比创造戏剧性 | 从窄走廊进入开阔大厅的震撼感 |
| **[[专业词汇注解/场所精神|场所精神]]（Genius Loci）** | 空间的独特情感氛围 | 搜打撤地图中废弃工厂的压抑感 vs 度假村的轻松感 |
| **[[专业词汇注解/前景与庇护|前景与庇护]]（Prospect & Refuge）** | 安全的观察点与可退守的遮蔽处 | 搜打撤中"窗口看走廊"（前景）、"角落打包"（庇护） |
| **分子空间（Molecule Spaces）** | 将空间类型视为节点和边 | 将每个 POI 看作"分子"，动线看作"化学键" |

**在搜打撤中的落地 —— 前景与庇护**：

搜打撤地图中，玩家交替处于"前景"（暴露、观察、射击）和"庇护"（隐蔽、恢复、决策）状态。好的关卡设计应该让这两种状态自然交替：

```
庇护（安全）→ 前景（观察）→ 庇护（决策）→ 前景（推进）→ 庇护（恢复）→ ...
```

在搜打撤中，"庇护"点还包括：打包位（安全恢复）、蹲守位（伏击）、撤退路线（放弃战斗）。

### 6. Steve Lee 的 [[专业词汇注解/G-E-G-Q-S|G-E-G-Q-S 框架]]

**来源**：Steve Lee，《耻辱》（Dishonored）关卡设计师，GDC 演讲。

**核心内容**：关卡前期规划的五个步骤：

```
Goals（目标）      → 玩家要做什么？为什么做？怎么做？
Elements（元素）   → 列出所有可交互/不可交互的物品
Gameplay Beats（节奏点） → 定义重要的玩法节点
Questions（问题）  → 站在玩家角度预判所有可能性
Sequence（序列）   → 列出玩家动线的完整流程
```

**在搜打撤中的落地**：

在设计一张搜打撤地图的核心 POI 时，用 G-E-G-Q-S 过一遍：

| 步骤 | 搜打撤示例（设计航天基地核心区） |
|------|-------------------------------|
| Goals | 玩家需要进入核心区搜刮高级物资，然后安全撤离 |
| Elements | 入口（刷卡/破门）、保险箱、服务器、通风管道、制高点 |
| Beats | 到达入口→刷卡/找钥匙→进入→清场→搜刮→被其他队伍发现→撤离 |
| Questions | 没卡怎么办？被架住入口怎么办？搜到顶级装备后怎么撤？ |
| Sequence | 出生点→判断核心区方向→接近→入口决策→进入→搜刮→警戒→撤离 |

### 7. 宫本茂的[[专业词汇注解/箱庭理论|箱庭理论]]

**来源**：宫本茂（任天堂），"箱庭"（盆栽式花园）的关卡设计哲学。

**核心内容**：在有限空间内最大限度地发挥游戏趣味性。箱庭的精髓是：
- **空间的浓缩**：每个区域都有密度适当的可交互内容
- **视点构成**：从特定角度看到的画面本身就是引导
- **地标的吸引力**：一个显眼的目标驱动玩家探索
- **"啊哈"时刻**：玩家自己发现隐藏路径/秘密时的惊喜

**在搜打撤中的落地**：

箱庭理论对搜打撤地图设计的一个关键启示：**好的搜打撤地图不是越大越好，而是密度恰到好处**。三角洲的零号大坝比航天基地小得多，但新手体验反而更好——因为空间紧凑，决策点密集，每一分钟都有事做。

**设计应用**："我在设计 POI 时会用箱庭思维的思路——每个区域都要有足够的内容密度，不能为了大而大。玩家走过 20 秒没有任何可交互内容，那就是应该砍掉的空间。"

## 五、实操建议

### 1. 用模板拆地图时的要点

当你用 `[[_模板_地图拆解]]` 做拆解时，重点关注：
- **不要只描述"有什么"**，要分析"为什么这么设计"
- 每个区域都问：**设计者想让我在这里获得什么体验？**
- 动线分析时**画出俯视图**，标注玩家可能的行动路线
- 如果发现设计问题，提出**你的修改方案**——这最能体现设计能力

### 2. 学习路径

| 阶段 | 时间 | 产出 |
|------|------|------|
| 理论预热 | 第 1 周 | 掌握核心概念，能复述设计流程 |
| 地图拆解 | 第 2-3 周 | 3-5 张地图的完整拆解报告 |
| 原创 Layout | 第 4-5 周 | 自己设计的搜打撤地图布局图 |
| UE5 白盒 | 第 6-8 周 | 可游玩的白盒原型（视频+截图） |

### 3. 日常工作里可以积累的

- 每次打三角洲/暗区/塔科夫，都带着"设计者视角"——注意动线选择、掩体位置、交战位置
- 记录你觉得"这个设计很好"和"这个设计很烂"的瞬间
- 截俯视图（无人机/上帝视角），画动线标注
- 练习在 10 分钟内快速画一张 Layout 草图

- **死区/过渡带** ^term-deadzone — Dead Zone，低物资、低掩体、高风险的过渡区域，迫使玩家快速通过或绕行

- **枪线** ^term-sightline — Sightline / Lane，玩家可以射击的视线通道，长枪线适合狙击，短枪线适合近战

- **掩体** ^term-cover — Cover，为战斗提供安全空间的设计元素
- **全掩体** ^term-fullcover — Full Cover，完全遮挡角色身体的掩体，可安全打包/换弹
- **半掩体** ^term-halfcover — Half Cover，部分遮挡的掩体，可架枪对枪但不能完全躲避
- **可穿透掩体** ^term-pencover — Penetrable Cover，子弹可以打穿的掩体（木门/薄墙），提供信息博弈空间
- **TTK** ^term-ttk — Time to Kill，击杀时间，从开始射击到目标死亡的时间，决定战斗节奏和掩体需求
- **交战距离** ^term-engagement — Engagement Range，战斗发生的距离范围，影响武器选择和空间设计
- **可供性** ^term-affordance — Affordance，物体的形状暗示其用途（如梯子暗示可攀爬）
- **否认可供性** ^term-negaffordance — Negative Affordance，用清晰障碍暗示"此路不通"
- **引导线** ^term-leadingline — Leading Line，道路、管道、电缆等延伸性元素引导视线至目标
- **面包屑引导** ^term-breadcrumb — Breadcrumbing，用环境细节（血迹、物资、尸体）逐步引导玩家深入

- **软相遇** ^term-softencounter — Soft Encounter，不是强制战斗而是"可能碰到人"的遭遇设计
- **硬性相遇** ^term-hardencounter — Hard Encounter，在 Choke/必经之路上的确定性遭遇
- **跑刀/裸跑** ^term-kniferun — Knife Run，零成本进局、只搜高价值物品的玩法
- **老鼠/鼠鼠** ^term-rat — Rat，避战搜刮、以积累经济为主的玩法流派
- **猛攻/大哥** ^term-chad — Chad，主动找架打、以战养战的玩法流派
- **生产者** ^term-producer — Producer，将局内物资带出并出售给市场的玩家（主要是老鼠）
- **消费者** ^term-consumer — Consumer，购买补给和高级装备的玩家（主要是猛攻）
- **安全期** ^term-safeperiod — Safe Period，进局前期相对安全、不会立即遭遇战斗的时间窗口

### 游戏模式与概念类

- **PvP** ^term-pvp — Player vs Player，玩家对抗玩家
- **PvE** ^term-pve — Player vs Environment，玩家对抗环境（AI 敌人）
- **PvPvE** ^term-pvpve — Player vs Player vs Environment，玩家同时对抗其他玩家和环境威胁，搜打撤的标准模式
- **FPS** ^term-fps — First-Person Shooter，第一人称射击游戏
- **TPS** ^term-tps — Third-Person Shooter，第三人称射击游戏
- **3C** ^term-3c — Character, Camera, Control，角色、视角、控制，决定游戏操作手感的三个核心要素

### 数据与测试类

- **热力图** ^term-heatmap — Heatmap，用颜色密度可视化玩家活动/死亡/击杀位置的图表
- **撤离率** ^term-extractrate — Extract Rate，成功撤离的对局比例
- **留存率** ^term-retention — Retention Rate，玩家在某个区域的停留/回访概率
- **分区留存率** ^term-zoneretention — Zone Retention，各区域玩家的停留时长分布
- **全队覆灭率** ^term-teamwipe — Team Wipe Rate，在某个区域全队阵亡的比例
- **跑测** ^term-playtest — Playtest，在实际可玩的版本中进行测试

### 经典理论类

- **心流理论** ^term-flowtheory — Flow Theory，Csikszentmihalyi 提出的心理状态，挑战与技能动态平衡时产生沉浸感
- **MDA 框架** ^term-mda — Mechanics-Dynamics-Aesthetics，将游戏拆解为机制→动态→美学三个层次的分析框架
- **城市意象五要素** ^term-lynch — Kevin Lynch 提出的路径/边界/区域/节点/地标五个空间认知要素
- **模式语言** ^term-patternlanguage — Christopher Alexander 的 253 个建筑空间设计模式
- **图底关系** ^term-figureground — Figure-Ground，建筑实体与开放空间的关系
- **场所精神** ^term-geniusloci — Genius Loci，空间的独特情感氛围和场所感
- **前景与庇护** ^term-prospectrefuge — Prospect & Refuge，安全的观察点（前景）与可退守的遮蔽处（庇护）交替的理论
- **G-E-G-Q-S** ^term-gegqs — Goals-Elements-Gameplay Beats-Questions-Sequence，Steve Lee 的关卡前期规划五步法
- **箱庭理论** ^term-sandbox2 — Sandbox Garden Theory，宫本茂的"有限空间内最大化趣味"设计哲学
