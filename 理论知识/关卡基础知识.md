# 关卡设计基础知识

## TL;DR
- 关卡设计的本质是**设计空间**——通过实体（几何体、建筑、物件）限定出有意义的空间，引导玩家体验
- 核心三要素：**布局**（空间怎么组织）、**动线**（玩家怎么走）、**节奏**（高潮低谷怎么排）
- 标准流程：概念草图 → 布局图 → 白模测试 → 脚本 → 美术 → 打磨，白模阶段决定了 80% 的成败
- 最好的学习方式是**做**——画图、搭白模、跑测试，别只读理论

---

## 1. 什么是关卡设计

### 1.1 核心定义

关卡设计的本质是**设计"空间"**。空间由实体（几何体、物件）限定和影响而诞生——在一片空白中摆上实体时，有意义的空间便被创造出来。

> 空间由**点、线、面**构成。这些特征越显著，空间的存在感越强。

一个好关卡同时是三样东西：**沉默的讲述者**（环境叙事）、**挑战**（玩法阻碍）、**游乐场**（自由探索）。

### 1.2 关卡设计师做什么

| 做什么 | 不做什么 |
|--------|---------|
| 设计空间布局和动线 | 写数值公式 |
| 控制节奏和玩家情绪 | 做美术资源 |
| 搭建白模并反复测试 | 写剧情文本 |
| 放置玩法元素（敌人、道具、机关） | 设计技能系统 |

### 1.3 空间设计的本质

- **积极空间**：有明确边界、中心性和几何结构的空间，人们倾向于在此停留
- **消极空间**：边界模糊、缺乏中心的空间，人们倾向于快速穿行
- **核心手法**：在重点区域强化积极空间特征（对称、围合、地标），让玩家在此停留并完成关键内容

## 2. 关卡设计的核心三要素

### 2.1 布局（Layout）——空间怎么组织

布局是关卡设计的骨架，用自上而下的 2D 俯视图呈现。

**布局图的绘制层级：**

```
概念草图（Parti）
    → 5-10 个核心形状的头脑风暴
    ↓
气泡图
    → 椭圆代表不同区域，标注空间关系和比例
    ↓
平面图
    → 详细标注墙壁、门、窗户等结构
    ↓
玩法标记
    → 标注路径、区域名、敌人、道具、陷阱
```

**布局的常见范式：**

| 类型 | 特点 | 适用场景 |
|------|------|---------|
| 线性 | 单一路径从头到尾 | 叙事驱动、新手教学 |
| 分支 | 多条可选路径 | 提供选择感 |
| 环形 | 路径回到起点形成循环 | 节省空间、增加探索深度 |
| 竞技场 | 一个或多个大型战斗空间 | Boss 战、防守关 |
| 枢纽-辐条 | 一个中心连接多个分支 | 开放世界、类银河城 |

### 2.2 动线（Flow）——玩家怎么走

动线的本质是**设计移动**——玩家在关卡中沿着不同路径移动的心理感受。

**三个核心概念：**

| 概念 | 说明 |
|------|------|
| **理想路径（Desire Lines）** | 玩家直觉上最想走的"自然路线" |
| **主线路径（Critical Path）** | 完成关卡的最小必经路径 |
| **流线（Circulation）** | 区域间的连接方式，分为叙事流线和形式流线 |

**引导玩家的工具：**
- 光线（亮处吸引注意）
- 颜色对比（重点区域用醒目颜色）
- 地标（远处可见的目标物，时隐时现最佳）
- 构图引导线（墙面、栏杆、地形的延伸方向）
- 敌人位置（敌人出现的方向暗示"那边重要"）
- 声音提示（远处枪声、机器声引导方向）
- 地形坡度（上坡暗示前进，下坡暗示返回）

> 最好的引导让玩家感觉是**自己发现了路线**，而非被牵着走。

**控制动线的工具：**
- **门/锁（Gates）**：一次性或条件性的阻挡，控制进度
- **阻塞点（Choke Points）**：收窄空间增加遭遇频率
- **捷径（Shortcuts）**：单向开启，方便回溯
- **死区（Dead Space）**：空旷无掩体的空间，玩家快速穿过

### 2.3 节奏（Pacing）——高潮低谷怎么排

节奏是关卡中活动与事件的顺序和情绪曲线。

**基本模式：「教学 → 测试 → 扭转」**

```
教学：安全环境介绍机制
  ↓
测试：在适度压力下要求玩家使用
  ↓
扭转：结合新条件进行变化，刷新体验
```

**节奏曲线示意：**

```
紧张度
  │    ╭──╮         ╭────╮
  │   ╱    ╲       ╱      ╲     ╭──
  │──╱      ╲─────╱        ╲───╱
  │                              ╲──
  └──────────────────────────────→ 时间
    探索  战斗  喘息  Boss  收尾
```

核心原则：**交替高潮与低谷**。连续战斗让玩家疲劳，连续探索让玩家无聊。

## 3. 关卡设计流程

### 3.1 标准七步流程

| 步骤 | 内容 | 产出物 |
|------|------|--------|
| **1. 前期策划** | 明确目标、玩法节点、玩家体验目标 | 设计文档（Goals/Beats/Questions） |
| **2. 概念草图** | 头脑风暴核心形状和布局思路 | Parti 草图 |
| **3. 布局图** | 细化 2D 俯视图，标注路径和区域 | 详细平面图 |
| **4. 白模** | 用简单几何体搭建可玩版本 | 引擎内可跑的灰盒 |
| **5. 脚本** | 添加门、按钮、AI、事件逻辑 | 可玩的完整关卡 |
| **6. 美术** | 替换为正式美术资源、灯光、材质 | 接近成品的关卡 |
| **7. 打磨** | 根据测试反馈修 Bug、调难度 | 发布版本 |

### 3.2 白模——最重要的一步

白模是用简单几何体（方块、坡道、圆柱等）搭建的关卡草稿，没有精致美术。

**为什么白模决定 80% 的成败：**
- 删改白模几何体几乎零成本，但废弃已完成的美术资源代价极高
- 只有白模可以真正**游戏测试**——图纸和文档做不到
- 在大量投入美术之前，先验证玩法

**搭建白模时审视四个维度：**
- 体量感——结构厚重还是轻薄？
- 尺度——区域太大还是太小？
- 路径引导——玩家能否找到出口？会不会迷路？
- 游戏测试——亲手跑一遍，别只在编辑器里飞

### 3.3 SCOTT 方法（实用启动框架）

| 步骤 | 动作 |
|------|------|
| **S**tart | 定义关卡目标、玩家做什么、该感受什么 |
| **C**larify | 用一两句话概括关卡核心意图——"北极星" |
| **O**h god | 收集参考：游戏、电影、现实建筑，拆解它们为什么好用 |
| **T**ry | 快速原型：最快速度把基础几何体扔进引擎跑起来 |
| **T**eam up | 尽早让别人测试——没上下文的人能看到你的盲区 |

## 4. 常见关卡类型

| 类型 | 核心特征 | 代表游戏 |
|------|---------|---------|
| **线性关卡** | 单一路径，严格控制节奏和教学 | 《半条命2》《神秘海域》 |
| **宽线性关卡** | 单一路径但每段有宽广空间可选路线 | 《最后生还者》 |
| **分支关卡** | 多条可选路径，不同选择带来不同体验 | 《耻辱》《生化危机》 |
| **枢纽式** | 一个中心安全区连接多个分支区域 | 《黑暗之魂》传火祭祀场 |
| **开放区域** | 大地图上散布 POI，玩家自由选择顺序 | 《艾尔登法环》宁姆格福 |
| **竞技场** | 封闭或半封闭的大型战斗空间 | 《毁灭战士》2016 |
| **模块化** | 可重复组合的标准化组件拼接 | 大部分 FPS 多人地图 |
| **程序生成** | 算法自动组合关卡元素 | 《以撒的结合》《死亡细胞》 |

## 5. 新人怎么入门

### 5.1 学习路径

```
理论入门 → 白模练习 → 拆解经典 → 做作品集
```

1. **理论入门**：读《The Level Design Book》、看 GDC 经典演讲
2. **白模练习**：在引擎中搭一个简单的 3D 空间，跑起来感受尺度
3. **拆解经典**：选一个喜欢的关卡，画它的布局图、标出动线和节奏
4. **做作品集**：2-3 个可玩关卡，附设计文档说明思考过程

### 5.2 最重要的习惯

- **画图**：动手之前在纸上画布局，比脑子里空想清晰得多
- **跑测**：每改一版就亲自跑一遍，别只在编辑器视角飞
- **看别人玩**：观察测试者在哪里犹豫、迷路、误解——这是最宝贵的反馈
- **收集参考**：截图、画草图、拆解——每个好关卡背后都有大量参考

### 5.3 推荐资源

| 资源 | 说明 |
|------|------|
