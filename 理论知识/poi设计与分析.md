# 搜打撤 POI 设计要点

## TL;DR
- 搜打撤 POI 设计的核心矛盾是"搜—打—撤"三者难以同时做好，POI 布局决定了玩家的节奏和博弈深度
- 宏观上通过多地图实现"经济漏斗"：低难图生产资源，高难图消耗资源
- 中观上动线由三组关系决定：复活→搜索、搜索→搜索、搜索→撤离
- 四款游戏风格差异明显：塔科夫重搜、三角洲重打、暗区重博弈、ARC Raiders 重 PvE 协作

---

## 1. 搜打撤 POI 的核心定位

### 1.1 搜打撤的三重张力

搜打撤游戏有一个内在矛盾——「搜」（自由选择区域搜索）、「打」（自由选择是否战斗）、「撤」（自由选择撤离方式）三者难以同时做好：提升"搜"会分散玩家导致"打"变弱，强化"打"则需要限制"搜"的区域收益。

POI 就是这个矛盾的**调节器**——通过控制资源分布来调控玩家的风险决策。

### 1.2 POI 在搜打撤中的独特角色

与传统大逃杀不同，搜打撤的 POI 不是"谁活到最后谁拿"，而是"谁拿到了能不能带出去"。这决定了 POI 设计需要考虑：
- **进入成本 vs 撤离成本**：高风险区不仅要考虑怎么进去，更要考虑怎么带着东西出来
- **时间窗口**：一个 POI 的价值随时间变化（别人先搜完就没了）
- **信息博弈**：枪声、拉闸声暴露位置，形成"信息代价"

## 2. POI 关卡布局的三组动线

搜打撤的 POI 布局可以从三组动线关系来拆解：

### 2.1 复活点 → 搜索点

| 模式 | 代表案例 | 特点 |
|------|---------|------|
| **先搜后打** | 三角洲大坝 | 2-3 复活点对应 1 个核心区，一主两次，初期有安全期 |
| **先打后搜** | 地铁逃生 | 唯一核心资源区 + 多个 Choke，逼玩家先打再搜 |
| **以事件换空间** | 三角洲巴克什（巴别塔） | 拉闸/刷卡触发，信息暴露换资源获取窗口 |
| **强制交战** | 三角洲航天 | 复活点远离核心区，Choke 收束动线 |

> 塔科夫：复活点通常在边缘，路径自由度极高，依赖玩家对地图的全局认知。

### 2.2 搜索点 → 搜索点

核心搜索区的**对称设计**是搜打撤的关键特征：
- 三角洲大坝东西行政楼单轴对称
- 三角洲航天 5 个搜索区两轴对称
- 塔科夫物资分布极精细——墙面、柜子、抽屉均可交互，搜索密度远超国产游戏

**随机事件创造动态 POI**：
- 三角洲通过随机任务（破壁、博士等）在核心区外围创造固定时间的博弈窗口
- 随机性与学习成本的平衡是高级设计要点

### 2.3 搜索点 → 撤离点

撤离点的**数量、位置、条件**是区分地图难度的最直接手段：

| 策略 | 案例 | 效果 |
|------|------|------|
| 多数量 / 低条件 | 三角洲大坝、长弓 | 服务新手，低挫败感 |
| 少数量 / 高条件 | 三角洲航天 | 3 个撤离点均需过桥，衍生堵桥玩法 |
| 拉闸撤离 | 三角洲高难图 | 拉闸 = 信息暴露 + 时间等待，位于核心区 |
| 付费原地撤离 | 地铁逃生 | 可购买电话原地撤，降低压力 |
| 高惩罚制 | 塔科夫 | 撤离点固定但需记忆位置，死亡全丢 |

## 3. 四款游戏 POI 设计对比

### 3.1 塔科夫——「搜」做到极致

| 维度 | 特点 |
|------|------|
| 搜索密度 | 极高。每栋建筑数十个可交互点，远超国产游戏 |
| 地图分类 | 对冲型（海关）vs 交汇型（其余），动线强制遭遇 |
| 撤离设计 | 死亡全丢，撤离点固定但随机分配，出生与撤离需横穿全图 |
| 经济控制 | 强制删档周期重置经济，设计大量"消耗点"延缓膨胀 |

**社区评价**：氛围和世界构建极强（环境叙事细节丰富），但 PvP 平衡争议大——设计偏向"强制冲突以消耗资源"而非策略性玩法。

### 3.2 三角洲行动——「打」做到极致

| 维度 | 特点 |
|------|------|
| 地图分级 | 大坝→长弓→巴克什→航天，四级递进 |
| 设计理念 | "以打为核心"，干员技能辅助猛攻 |
| 容器体系 | 明确的大保/小保/任务 POI，搜索中等密度 |
| 撤离设计 | 安全箱保底 + 多种撤离选择，压力适中 |
| 随机事件 | 破壁、博士等任务在核心区外围创造动态博弈点 |

**核心设计**：四级地图形成经济漏斗——低难图生产资源，高难图消耗资源。

### 3.3 暗区突围无限——「博弈」做到极致

| 维度 | 特点 |
|------|------|
| 搜索密度 | 中等偏低，搜索非核心 |
| 设计理念 | 信息收集与博弈为主 |
| 难度分层 | 普通 / 封锁区（10-15万装备）/ 禁区（30万装备）三级 |
| 撤离设计 | 条件丰富——电梯（需拉双开关）、排水管（不能带背包）、直升机（需狗牌） |
| 地图特点 | 机场图采用环形结构（内圈/中圈/外圈），服务不同风险偏好的玩家 |
| 动态元素 | Season 4 加入随机刷新保险箱，创造动态 loot 路径 |

**行业说法**："暗区没有搜"——自由搜索被弱化，重心在信息对抗。

### 3.4 ARC Raiders——「协作」做到极致

| 维度 | 特点 |
|------|------|
| 搜索密度 | 中高——抽屉、柜子、衣柜等可交互容器丰富 |
| 设计理念 | PvPvE，PvE 占比极大，机器人敌人是核心威胁 |
| 地图特点 | 4 张首发地图，无小地图，30 分钟倒计时核爆强制撤离 |
| 撤离设计 | 货运电梯（10 秒激活 + 大声蜂鸣暴露位置）、舱门钥匙、火车 |
| 免费配装 | 仓库空了送基础枪+盾+背包，门槛极低 |
| 社交系统 | 近距离语音 + 表情，可临时组队背叛，是核心体验 |

**问题**：部分地图（Buried City）过度偏袒伏击者，Spaceport 过大过空缺乏掩体；后期社区"老鼠化"（蹲撤离点、先手开枪）影响体验。

---

## 4. 总结：POI 设计的核心控制变量

| 变量 | 作用 | 塔科夫 | 三角洲 | 暗区 | ARC Raiders |
|------|------|--------|--------|------|-------------|
| 搜索密度 | 决定搜的深度 | 极高 | 中 | 中低 | 中高 |
| Choke 数量 | 调控遭遇频率 | 多 | 中 | 中 | 少 |
| 撤离难度 | 调控风险压力 | 极高 | 中 | 中 | 低（免费配装兜底）|
| 事件驱动 | 创造动态博弈 | 少 | 多 | 中 | 天气/时间/敌人巡逻 |
| PvE 占比 | 分流玩家对抗 | 低 | 低 | 低 | **极高** |
