# 搜打撤动线设计参考

> 动线（Flow / Circulation）是搜打撤关卡设计的**骨骼**——它决定了玩家在哪相遇、何时战斗、如何撤离。
>
> 本文聚焦于搜打撤品类中动线设计的独特问题：如何在"自由探索"和"可控遭遇"之间取得平衡。

---

## 一、什么是动线

**动线** = 玩家在关卡中的移动轨迹和路径网络。它是空间布局的"流通系统"。

### 1.1 搜打撤动线的特殊性

与传统 FPS 的动线设计相比，搜打撤的核心差异在于：

| 维度 | 传统 FPS（CS/CoD） | 搜打撤 |
|------|-------------------|--------|
| **路径自由度** | 低——固定进攻/防守路线 | 高——玩家自主选择去哪、打不打 |
| **遭遇可预测性** | 高——对称布局确保遭遇 | 低——随机出生 + 自由路线 |
| **动线目标** | 到达目标点/击杀敌人 | 搜刮物资 + 安全撤离 |
| **设计难点** | 路线平衡（50/50 胜率） | 遭遇密度控制（不要太密也不要太稀） |
| **节奏控制者** | 设计师主导（脚本事件） | 玩家自调节（选择挑战水平） |

> **核心矛盾**：搜打撤的动线需要同时服务「搜」（自由探索）、「打」（可控遭遇）、「撤」（紧张收束）三个互相冲突的目标。

### 1.2 动线设计的三层结构

```
┌──────────────────────────────────────┐
│  欲望路径（Desire Lines）              │  ← 玩家本能想走的最自然路线
│  ┌────────────────────────────────┐  │
│  │  黄金路径（Critical Path）       │  │  ← 完成关卡的最小必经路径
│  │  ┌──────────────────────────┐  │  │
│  │  │  流线（Circulation）      │  │  │  ← 区域之间的连接方式网络
│  │  └──────────────────────────┘  │  │
│  └────────────────────────────────┘  │
└──────────────────────────────────────┘
```

| 层次 | 定义 | 设计用途 |
|------|------|---------|
| **[[专业词汇注解/欲望路径\|欲望路径]]** | 玩家本能想走的最自然路线 | 对比"规划路径"与"实际路径"的差距，发现设计问题 |
| **[[专业词汇注解/黄金路径\|黄金路径]]** | 完成关卡的最小必经路径 | 确保主线体验完整，识别哪些内容是可选的 |
| **[[专业词汇注解/流线\|流线]]** | 区域间的连接方式网络 | 宏观组织空间关系，决定遭遇概率分布 |

## 二、动线的基本构成元素

一张搜打撤地图的动线网络由以下元素构成：

### 2.1 元素全景

```
                         ┌─────────────┐
                         │   撤离点     │
                         │  (Extract)  │
                         └──────┬──────┘
                                │
    ┌──────────┐          ┌─────┴─────┐          ┌──────────┐
    │  出生点A  ├──主干道──→│ 高价值POI  │←──主干道──┤  出生点B  │
    └──────────┘          └──┬───┬───┬─┘          └──────────┘
                             │   │   │
                      ┌──────┘   │   └──────┐
                      ↓          ↓          ↓
                  侧翼路      Choke点      死区
                 (绕行)     (必经之路)   (快速通过)
                      │          │          │
                      └──────────┼──────────┘
                                 ↓
                           下一个 POI
```

### 2.2 各元素详解

| 元素 | 定义 | 动线中的作用 | 设计要点 |
|------|------|-------------|---------|
| **[[专业词汇注解/主干道\|主干道]]** | 连接主要区域的核心路线 | 大部分玩家进攻/转移的主要通道 | 足够空间容纳战斗 + 适当掩体 + 清晰视觉引导 |
| **[[专业词汇注解/侧翼路\|侧翼路]]** | 用于绕后或避让的辅助路线 | 提供战术选择，避免主干道被完全封锁 | 到达目标的时间成本 > 主干道（以时间换安全） |
| **[[专业词汇注解/Choke点\|Choke]]** | 动线收束的狭窄区域 | 制造确定性遭遇，控制战斗节奏 | 双方距离对等 + 至少一条侧翼 + 投掷物空间 |
| **[[专业词汇注解/死区\|死区]]** | 低物资、低掩体的过渡区域 | 迫使玩家快速通过，制造紧张感 | 位于 POI 之间的必经通道，不能让人想逗留 |
| **[[专业词汇注解/垂直转换点\|垂直转换点]]** | 连接不同高度层的节点 | 改变战斗维度，增加战术深度 | 楼梯/电梯/通风管/绳索，标注在 Layout 上 |

### 2.3 动线元素的组合逻辑

```
主干道 ──→ 高流量、高遭遇概率、可预测
侧翼路 ──→ 低流量、低遭遇概率、高安全性
Choke   ──→ 强制收束、遭遇确定性高
死区    ──→ 低停留意愿、快速通过、易被伏击
```

> **设计铁律**：主干道承担 60-70% 的流量，侧翼路承担 20-30%，死区承担 5-10%。如果侧翼路的使用率超过主干道，说明主干道设计出了问题（太危险/太绕/无价值）。

## 三、搜打撤三阶段动线模型

搜打撤的核心循环「搜索→战斗→撤离」中，每个阶段的动线关系完全不同。

### 3.1 三阶段全景

```
阶段 1: 出生点 → 搜索点        阶段 2: 搜索点 → 搜索点       阶段 3: 搜索点 → 撤离点
┌─────────────────┐      ┌─────────────────┐      ┌─────────────────┐
│ [出生A] ↘        │      │  [次级POI] →     │      │  [核心区] ↘     │
│          ↘       │      │      ↓           │      │           ↘     │
│ [出生B] → [POI]  │      │  [核心POI] ←     │      │  [撤离点A] (边缘)│
│          ↗       │      │      ↑           │      │           ↗     │
│ [出生C] ↗        │      │  [分散点] →      │      │  [撤离点B] (条件)│
└─────────────────┘      └─────────────────┘      └─────────────────┘
   节奏：探索型              节奏：战斗型              节奏：收束型
   遭遇：软相遇为主          遭遇：软+硬混合            遭遇：硬性相遇为主
```

### 3.2 阶段 1：出生点 → 搜索点

**核心决策**：先打后搜 vs 先搜后打

| 模式 | 动线特征 | 玩家体验 | 代表案例 |
|------|---------|---------|---------|
| **[[专业词汇注解/先打后搜\|先打后搜]]** | 出生点→Choke→核心区，必经 Choke | 早期接敌概率高，"想搜先打过我" | 地铁逃生 |
| **[[专业词汇注解/先搜后打\|先搜后打]]** | 出生点→安全期搜刮→逐渐收束 | 有安全期，战斗在资源区内发生 | 三角洲零号大坝 |
| **以事件换空间** | 拉闸/刷卡触发，暴露信息换资源窗口 | 主动选择何时暴露自己 | 三角洲巴克什（巴别塔） |
| **强制交战** | 出生点远离核心区，Choke 收束动线 | 高概率早期遭遇，高风险高回报 | 三角洲航天基地 |

**设计要点**：
- 相邻出生点之间应有 ≥ 20 秒路程的间隔
- 每个出生点至少 2 条不同方向的出口路线
- 劣势出生点需要补偿机制（更近的物资点/更早的动权）

### 3.3 阶段 2：搜索点 → 搜索点

**核心决策**：集中连接 vs 分散间隔

| 策略 | 动线特征 | 体验效果 |
|------|---------|---------|
| **集中连接** | POI 之间通过少数 Choke 连接 | 高遭遇概率，适合"打"为核心的体验 |
| **分散间隔** | POI 之间多条路线可选 | 低遭遇概率，适合"搜"为核心的体验 |
| **环形结构** | POI 围绕中心排列，形成内圈/中圈/外圈 | 不同风险偏好玩家走不同圈层（暗区机场图） |
| **对称布局** | 核心 POI 对称分布，双轴/单轴对称 | 公平性优先，抵消出生点优劣（三角洲航天） |

**设计要点**：
- POI 间距应与对局时长匹配——太近则全程连战，太远则跑图时间过长
- [[专业词汇注解/POI引力陷阱|POI 引力陷阱]]——避免单一 POI 过强导致所有玩家涌向同一区域
- 随机事件（如三角洲的破壁任务、暗区的随机保险箱）在核心区外围创造**动态动线**，打破固定路径依赖

### 3.4 阶段 3：搜索点 → 撤离点

**核心设计理念**：撤离点不是让你轻松走掉，而是**制造最后的博弈空间**。

| 撤离类型 | 位置 | 动线效果 |
|---------|------|---------|
| **常规边缘撤离** | 地图边缘 | 迫使玩家横穿全图，制造后期遭遇 |
| **条件撤离（拉闸/任务）** | 核心区附近 | 全图广播位置，信息风险极高 |
| **随机撤离** | 每局随机刷新 | 打破固定路径依赖，增加不可预测性 |
| **付费原地撤离** | 任意位置（消耗资源） | 提供"花钱买安全"的选项 |

**设计要点**：
- 撤离点的等待时间 = 战斗窗口——越长风险越大
- 一个好的撤离点让玩家面临三选一：
  1. 直接冲——高风险，可能被堵
  2. 先清场再撤——安全，但可能错过窗口
  3. 换另一个更安全的撤离点——绕路，路上可能遭遇

## 四、动线网络的拓扑设计

### 4.1 五种基本拓扑结构

| 类型 | 结构示意 | 遭遇可预测性 | 适用场景 |
|------|---------|-------------|---------|
| **线性** | A→B→C→D | 极高——所有人都沿同一路线 | 竞速模式、新手教学 |
| **分支** | A→B→{C, D} | 中——分叉后各自发展 | 提供选择感的中型图 |
| **环形** | A→B→C→A | 中——可能正面遭遇也可能错过 | 节省空间、增加探索深度 |
| **枢纽-辐条** | {A,B,C}→Hub→{D,E,F} | 高——Hub 是天然 Choke | 高价值中心区 + 分散出生点 |
| **网状** | 多点互联 | 低——路线高度自由 | 塔科夫式的硬核自由探索 |

### 4.2 搜打撤中的混合拓扑

实际搜打撤地图通常是**混合拓扑**——不同阶段使用不同结构：

```
阶段1（出生→搜索）：枢纽-辐条——多个出生点辐射到一个/多个 POI
阶段2（搜索→搜索）：网状或环形——POI 之间自由移动
阶段3（搜索→撤离）：线性收束——所有路线汇聚到少数撤离点
```

**核心设计原则**：自由度从高到低逐步收束——开局最自由（多条路线），终局最收束（少数撤离点）。

### 4.3 遭遇类型与动线的配合

动线设计本质上是在控制**遭遇类型**的分布：

| 遭遇类型 | 动线条件 | 玩家体验 |
|---------|---------|---------|
| **[[专业词汇注解/软相遇\|软相遇]]** | 多条路线可选，Choke 非必经 | "可能碰到人也可能碰不到"——不确定性带来紧张感 |
| **[[专业词汇注解/硬性相遇\|硬性相遇]]** | 单一路线 / 必经 Choke | "这里一定有人"——确定性带来备战心态 |

**分布原则**：
- 阶段 1（搜索）：软相遇为主（80%+），给玩家选择空间
- 阶段 2（战斗）：软+硬混合（各 50%），Choke 收束制造确定性战斗
- 阶段 3（撤离）：硬性相遇为主（70%+），撤离点周边形成天然 Choke

> **关键认知**：硬性相遇过多会让地图"公式化"——玩家知道每个拐角后都有人，失去探索感和紧张感。好的搜打撤地图是用软相遇制造"随时可能有人"的持续紧张，而不是用硬性相遇制造"这里一定有人"的可预测战斗。

## 五、动线的节奏控制

### 5.1 动线节奏的四个要素

| 要素 | 对应动线设计 | 控制手段 |
|------|------------|---------|
| **速度** | 玩家在动线上的移动快慢 | 地形（开阔→快，室内→慢）、威胁（敌人→慢） |
| **密度** | 单位时间内的遭遇/事件数量 | Choke 数量、POI 间距、路线数量 |
| **方向** | 玩家自然被引导的方向 | 地标、引导线、逐渐收束的空间 |
| **张力** | 动线的风险-回报梯度 | 主干道（高风险高回报）vs 侧翼路（低风险低回报） |

### 5.2 动线节奏曲线

理想的搜打撤动线节奏应呈现**渐进收束**：

```
自由度
  ↑
  │ ╲
  │  ╲           ← 阶段1：高自由度，多路线，软遭遇
  │   ╲
  │    ╲___      ← 阶段2：中等收束，Choke出现，遭遇密度上升
  │        ╲___
  │            ╲ ← 阶段3：强收束，少数撤离路线，遭遇密度峰值
  └──────────────────────────→ 对局时间
```

**反例**：
- ❌ 开局就强收束 → 出生杀，挫败感强
- ❌ 全程高自由度 → 碰不到人，无聊，"搜完就撤"
- ❌ 收束跳跃 → 从安全区直接进入死亡区，感觉不公平

### 5.3 紧张与放松的交替

动线上应交替出现"紧张段"和"放松段"，维持心流：

```
动线位置： 出生点 → 主干道 → POI外围 → POI内部 → 侧翼路 → 撤离点
           │        │         │          │         │        │
紧张度：   低       中        中-高      极高      低-中    极高
           │        │         │          │         │        │
玩家状态： 规划     警觉      观察       战斗      恢复     最后博弈
```

> 参见 [[心流在搜打撤关卡设计中的应用|心流理论]]中关于心流与动线节奏的详细分析。

## 六、动线与空间元素的配合

动线不是独立存在的——它必须与掩体、视野、垂直空间协同设计。

### 6.1 动线 × 掩体

| 场景 | 动线位置 | 掩体需求 |
|------|---------|---------|
| **主干道** | 高流量通道 | 两侧对称掩体，支持拉锯战 |
| **侧翼路** | 低流量绕行路线 | 少量掩体即可，但需覆盖关键拐角 |
| **Choke 入口** | 动线收束处 | 到达方和防守方都有掩体覆盖 |
| **死区** | 低掩体、高暴露 | 故意不给掩体——迫使快速通过 |
| **撤离点等待区** | 最后博弈空间 | 撤离方有部分掩体，拦截方有视野优势 |

### 6.2 动线 × 视野

在 Layout 阶段标注每条动线上的**枪线长度**和**视野盲区**：

```
动线 A（主干道）：
  [出生点] ──50m开阔── [掩体群] ──15m转角── [POI入口]
     │                    │              │
   长枪线                中距             CQB
   (狙击位)              (步枪位)        (近战位)
```

**原则**：一条动线上不应全程同一种交战距离——混合短/中/长距离才能服务不同武器和玩法风格。

### 6.3 动线 × 垂直空间

| 垂直关系 | 动线处理方式 |
|---------|------------|
| **室内 → 室外** | 门/窗作为转换点，入口处给掩体防"门口蹲" |
| **低地 → 高地** | 多条上坡路线，避免唯一楼梯被架死 |
| **屋顶 → 地面** | 屋顶给视野优势，但撤离路线有限（平衡风险） |
| **地下 → 地面** | 地下出口给掩体覆盖，防止被地面玩家守株待兔 |

### 6.4 动线上的"决策点"设计

在动线上每隔一段距离设置一个**决策点**——让玩家停下来做选择的位置：

| 决策点类型 | 动线上的位置 | 玩家面临的选择 |
|-----------|------------|--------------|
| **路线分叉** | 主干道与侧翼路分叉处 | "走大路快但危险，还是走小路安全但慢？" |
| **声源判断** | 听到枪声/脚步声时 | "过去看看能不能捡漏？还是避开？" |
| **资源取舍** | 经过次级 POI 时 | "花时间搜这个，还是直接去核心区？" |
| **撤离抉择** | 接近撤离点时 | "直接撤还是再搜一个点？这个撤离点安全吗？" |

> **设计原则**：决策点密度在阶段 1（高自由度）→ 阶段 3（低自由度）逐渐降低。好的搜打撤动线让玩家不断面临"有意义的选择"，而不是沿预设路径无脑走。

## 七、经典理论中的动线视角

### 7.1 凯文·林奇五要素 × 动线

| 要素 | 动线设计中的应用 |
|------|----------------|
| **路径** | 动线本身就是"路径"——确保每条路径方向感明确 |
| **边界** | 用边界限制动线选择范围，引导收束方向 |
| **区域** | 不同区域的动线"性格"应不同（室内=蜿蜒紧张，室外=开阔自由） |
| **节点** | 动线交汇处=决策点=Choke/十字路口 |
| **地标** | 沿动线的可见地标帮助玩家定位——"沿着能看到发射塔的路走" |

### 7.2 模式语言 × 动线

| 模式 | 动线中的应用 |
|------|------------|
| **路径与目标** | 沿动线始终能看到下一个目标点（但需要走过去才能到达） |
| **私密梯度** | 动线从公共→半公共→私密递进（开阔道路→POI外围→建筑内部） |
| **活动节点** | POI 间距 ≈ 玩家在单个 POI 的停留时长 × 移动速度 |

### 7.3 箱庭理论 × 动线

宫本茂的箱庭思维在动线设计中的应用：
- 每条动线都在**有限空间内提供密度适当的内容**
- 从特定视点看到的画面本身就是引导（"看到那边有个好东西，怎么过去？"）
- **"啊哈"时刻**：玩家自己发现隐藏的侧翼路线/近道时的惊喜

## 八、竞品动线设计对比

| 维度 | 塔科夫 | 三角洲行动 | 暗区突围无限 |
|------|--------|-----------|------------|
| **动线自由度** | 极高——网状拓扑，几乎全开放 | 中——枢纽-辐条为主，有明确主干 | 中——环形结构分层 |
| **动线引导方式** | 极隐晦——依赖玩家记忆和地标 | 适中——地标 + 任务引导 | 适中——环形分层自然引导 |
| **侧翼路丰富度** | 极高——大量可选路线 | 中——主要 POI 有 1-2 条侧翼 | 中——环形结构天然提供选择 |
| **Choke 数量** | 多——强制冲突以消耗资源 | 中——关键位置收束 | 中——环形层次间的转换点 |
| **撤离收束强度** | 极高——死亡全丢，撤离点固定 | 中——安全箱保底 + 多种选择 | 中——条件丰富（电梯/排水管/直升机） |
| **动线设计哲学** | "真实存在的地点" | "以打为核心的游戏场" | "信息博弈的棋盘" |

> 参见 [[理论知识/poi设计与分析|POI 设计要点]]中四款游戏的完整对比。

## 九、动线设计检查清单

### 9.1 Layout 阶段检查

- [ ] 每个出生点有 ≥ 2 条不同方向的出口路线
- [ ] 相邻出生点之间间距 ≥ 20 秒路程
- [ ] 存在清晰的主干道连接出生点→POI→撤离点
- [ ] 每个关键 Choke 至少有一条侧翼路可以绕行
- [ ] 撤离点数量与地图难度匹配（低难 4-6 个，高难 2-3 个）
- [ ] 三阶段动线关系已明确标注（出生→搜索、搜索→搜索、搜索→撤离）

### 9.2 遭遇设计检查

- [ ] 阶段 1（出生→搜索）软相遇占比 ≥ 80%
- [ ] 阶段 2（搜索→搜索）软+硬混合，不过度收束
- [ ] 阶段 3（搜索→撤离）逐渐收束，撤离点周边有自然 Choke
- [ ] 全图不存在"不遭遇走廊"——每条路线至少有一个可能的遭遇点
- [ ] 也不存在"必死走廊"——任何路线都不应完全无掩体暴露

### 9.3 节奏检查

- [ ] 自由度从高到低渐进收束（不是跳跃）
- [ ] 紧张段和放松段交替出现
- [ ] 沿动线的决策点密度合理（阶段 1 最高，阶段 3 最低）
- [ ] 玩家步行 20 秒内有明确的可交互内容或决策点
- [ ] 不存在无法安全通过的"绝对死区"

### 9.4 可读性检查

- [ ] 主干道有清晰的视觉引导（地标/光线/引导线）
- [ ] 关键决策点（分叉/Choke/入口）在视觉上可被识别
- [ ] 侧翼路不会让玩家误以为是主干道
- [ ] 死区有明显的"你不想在这里逗留"的视觉信号

## 十、动线设计的常见误区

| 误区 | 表现 | 修正方案 |
|------|------|---------|
| **对称至上** | 盲目追求动线对称，忽略体验 | 搜打撤不需要 50/50 平衡——需要的是"每个出生点都有合理的机会" |
| **路线过多** | 到处都是路，玩家永远碰不到人 | 减少可选路线，在关键位置收束 |
| **路线过少** | 唯一路线被完全压制，无解 | 增加侧翼路，确保每个 Choke 都有替代方案 |
| **收束过早** | 开局即遭遇，出生杀频发 | 延长安全期，出生点周边给更多路线选择 |
| **收束过晚** | 全程碰不到人，搜完就撤 | 在阶段 2 增加 Choke 密度，减少完全安全的路线 |
| **死区过大** | 长时间跑图无事发生 | 缩小死区尺寸，或增加分散的次级 POI 填充 |
| **撤离点无博弈** | 撤离点太安全，后期无紧张感 | 撤离点周边收束动线，减少掩体，增加等待时间 |
